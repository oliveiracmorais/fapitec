PROJETO FAPITEC

Tela inicial

1. Requisitos Funcionais
1.1 Autenticação

e

O usuário deve Inserir CPF ou Passaporte e Senha para acessar o sistema.

e

Deve haver validação de CPF no formato XXX. XXX. XXX-XX ou Passaporte
com letras e números.

e

À senha deve ter no mínimo 8 caracteres, incluindo letras, números e símbolos.

e

Deve haver um botão "Entrar" para enviar as credenciais ao servidor.

e

Caso os dados estejam incorretos, o sistema deve exibir uma mensagem de erro:
o

"CPF/Passaporte ou Senha inválidos. Tente novamente.”

1.2 Recuperação de Senha
e

Deve haver um botão "Esqueci minha senha” que leva o usuário a uma página
para redefinir sua senha.

e

O sistema deve permitir a redefinição via e-mail cadastrado ou código enviado
por SMS.

1.3 Cadastro de Novo Usuário
e

O link “clique aqui” deve direcionar para a página de cadastro.

e

No cadastro, devem ser exigidos CPF/Passaporte, Nome Completo, E-mail, Senha
e Confirmação de Senha.

e

O sistema deve validar se o CPF já existe antes de criar um novo cadastro.

1.4 Segurança

A senha deve ser armazenada de forma segura usando hashing com bcrypt ou
Argon2.

Após 5 tentativas falhas de login, o usuário deve ser bloqueado temporariamente
por 15 minutos.
Implementar Captcha caso sejam detectadas tentativas excessivas de login.
O sistema deve usar HTTPS para criptografar a comunicação.

2. Requisitos não Funcionais
2.1 Design e Responsividade

À tela deve ser responsiva para funcionar em desktop, tablets e smartphones.
O layout deve manter a identidade visual da FAPITEC/SE, incluindo cores, fontes
e logotipo.
Deve ser utilizado um framework moderno como Bootstrap ou Tailwind CSS.
2.2 Performance

O tempo de resposta para autenticação deve ser inferior a 2 segundos.
Às Imagens e ícones devem ser otimizados para carregamento rápido.
2.3 Acessibilidade
Os campos de entrada devem ter rótulos descritivos e placeholders.
Deve haver suporte para teclas de atalho e leitores de tela.
O contraste do texto deve seguir as diretrizes WCAG 2.1 para acessibilidade.
3. APT e Banco de Dados

3.1 Endponnts da API
POST login — Autenticação do usuário.

POST /reset-password — Envia link de recuperação por e-mail.
POST /register — Criação de novos usuários.
GET /user-profile — Retorna informações do usuário autenticado.
