# Story 1.17: Email para Recuperacao de Senha

## Status

Ready for Review

## Story

**Como** usuario da plataforma da FAPITEC-SE,
**quero** receber um email com link para redefinir minha senha,
**para que** o fluxo de recuperacao de senha seja completo de ponta a ponta sem intervencao manual.

## Acceptance Criteria

1. O adaptador de email deve enviar SMTP real quando configurado (fallback para log se nao configurado).
2. As variaveis de ambiente `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, `SMTP_FROM` devem controlar o envio.
3. O email deve conter um link HTML para `{FRONTEND_URL}/redefinir-senha?token={token}`.
4. Deve existir uma pagina frontend em `/redefinir-senha` com formulario de token + nova senha + confirmacao.
5. O token deve ser hasheado (SHA-256) antes de persistir no banco; o link do email carrega o token puro.
6. A pagina de redefinicao deve chamar `POST /api/v1/redefinir-senha` e exibir mensagem de sucesso/erro.
7. O wiring no `main.go` deve escolher condicionalmente entre adapter SMTP e adapter Log.
8. A interface `ServicoDeEmail` nao deve ser modificada (nenhuma quebra no dominio).
9. O fluxo completo deve ser testado: solicitacao → recebimento do email → clique no link → redefinicao → login com nova senha.
10. A story deve manter checklist e file list atualizados.

## Dev Notes

- Fonte primaria: Story 1.11 (recuperacao de senha) — o caso de uso ja esta completo, falta apenas o adapter de email e a pagina de redefinicao
- Usar `net/smtp` da stdlib Go — zero dependencias externas
- Template HTML de email: usar string template simples com cores institucionais da FAPITEC-SE
- Hashing do token: `sha256.Sum256([]byte(token))` antes de persistir; o token puro vai apenas no link do email
- `FRONTEND_URL`: nova env var ou usar `NEXT_PUBLIC_API_URL` como base
- Para testes locais de SMTP, recomendar Mailtrap (smtp.mailtrap.io)
- Nao quebrar a interface `ServicoDeEmail` do dominio — o adapter SMTP implementa a mesma interface
