# Story 1.11: Recuperação de Senha

## Status

Ready for Review

## Story

**Como** usuário da plataforma da `FAPITEC-SE`,
**quero** recuperar minha senha quando esquecê-la,
**para que** eu possa restabelecer meu acesso ao sistema sem intervenção manual da equipe administrativa.

## Acceptance Criteria

1. A tela de login deve conter um botão "Esqueci minha senha" que redireciona para o fluxo de recuperação.
2. O sistema deve permitir redefinição de senha via e-mail cadastrado como fluxo base.
3. O sistema deve enviar um código ou link de redefinição para o e-mail do usuário.
4. O fluxo de recuperação não deve informar se o e-mail/CPF existe ou não (proteção contra enumeração de usuários).
5. A recuperação via SMS deve ser prevista como capacidade futura, sem bloquear a arquitetura.
6. A solicitação de recuperação deve gerar evento de auditoria compatível com a Story 1.5.
7. A redefinição de senha deve exigir confirmação da nova senha.
8. Tokens de redefinição devem expirar após período definido (ex: 1 hora).
9. A story deve manter checklist e file list atualizados.
