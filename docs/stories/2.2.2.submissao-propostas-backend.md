# Story 2.2.2: Submissão de Propostas — Backend

## Status

Concluída (backend completo)

## Story

**Como** proponente da FAPITEC-SE,
**quero** submeter propostas de pesquisa vinculadas a editais vigentes,
**para que** minha instituição possa concorrer aos recursos de fomento.

## Context

O domínio `inscricao_e_selecao_de_projetos` está vazio. O Termo de Referência (seção 4.2) especifica ambiente para submissão de propostas com upload de documentação e validação automática. O ProjetoFapitec.txt detalha o fluxo completo: cadastro do proponente (dados pessoais, formação, documentação, empresa), planilha orçamentária, geração de protocolo e acompanhamento de status.

Esta story implementa o backend completo do domínio: entidades, value objects, casos de uso, repositórios e endpoints REST.

## Acceptance Criteria

1. O sistema deve permitir submeter proposta vinculada a um edital ativo.
2. A proposta deve conter: dados do proponente (pessoa física), dados acadêmicos, documentos digitais, vínculo empresarial (se exigido pelo edital), planilha orçamentária (itens de consumo e permanentes).
3. O sistema deve atribuir um protocolo único à proposta no momento da submissão.
4. O sistema deve validar: edital ativo, prazo dentro da vigência, campos obrigatórios preenchidos, documentos exigidos pelo edital anexados.
5. O proponente pode editar a proposta enquanto o edital estiver vigente.
6. O sistema deve manter histórico de versões das alterações.
7. O sistema deve listar propostas por proponente e por edital.
8. O orçamento deve ter valor total calculado automaticamente (quantidade × valor unitário) e respeitar o valor global do edital.
9. Os valores monetários devem usar representação inteira em centavos.
10. A implementação deve seguir DDD + Clean Architecture, mesmo padrão dos módulos existentes.

## Dev Notes

- Protocolo: formato `YYYY.EDITALID.SEQ` (ex.: `2026.5.001`)
- Status inicial é `rascunho`; muda para `submetida` após ação explícita de submeter
- Validação de orçamento contra valor global do edital deve usar os centavos (int64)
- Upload de documentos: nesta story, salvar em disco local (`/data/documentos/`); evoluir depois
- Reaproveitar padrão de `identidade_e_acesso` para estrutura de pastas
- Endpoints devem respeitar autorização por perfil (admin vê todas, proponente só as suas)
