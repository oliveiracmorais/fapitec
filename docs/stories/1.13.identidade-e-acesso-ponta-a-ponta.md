# Story 1.13: Integração Ponta-a-Ponta do identidade_e_acesso

## Status

Ready for Review

## Story

**Como** equipe de desenvolvimento da FAPITEC-SE,
**quero** conectar o frontend web ao backend Go do módulo `identidade_e_acesso`,
**para que** o primeiro módulo esteja totalmente funcional e valide o pipeline completo (Next.js → Go → PostgreSQL).

## Acceptance Criteria

1. A página inicial deve exibir um formulário de login real (CPF + Senha) conectado à API.
2. Deve existir uma página de cadastro de usuário com todos os campos obrigatórios.
3. O fluxo de recuperação de senha deve estar funcional (já implementado).
4. Após login, o usuário deve ser redirecionado a um dashboard com seus dados.
5. O dashboard deve exibir nome, documento e e-mail do usuário autenticado.
6. A sessão deve ser mantida via token/session storage no navegador.
7. Tailwind CSS deve ser instalado e configurado como framework de estilização.

## Tasks / Subtasks

- [x] Instalar e configurar Tailwind CSS no frontend
- [x] Criar página de login real (`/`) com formulário CPF + Senha
- [x] Criar página de cadastro real (`/cadastro`)
- [x] Criar página de dashboard pós-login (`/dashboard`)
- [x] Implementar sessão via localStorage

## Dev Notes

- Fonte primária: Projeto FAPITEC (seções 1.1 Autenticação e 1.3 Cadastro)
- Backend já expõe: `POST /api/v1/login`, `POST /api/v1/cadastro`, `POST /api/v1/solicitar-redefinicao-senha`
- Proxy Next.js já configurado em `next.config.mjs`
- Login retorna: `{id, nome, documento, email, estrangeiro}`
- Cadastro retorna: `{id, nome, documento, email, estrangeiro}`
- Usar Tailwind CSS (padrão do projeto) sem dependência de shadcn/ui nesta fase
- Sessão: armazenar dados do usuário + timestamp no localStorage
