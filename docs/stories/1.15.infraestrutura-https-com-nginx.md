# Story 1.15: Infraestrutura HTTPS com nginx

## Status

Em Planejamento

## Story

**Como** equipe de infraestrutura da FAPITEC-SE,
**quero** configurar HTTPS no ambiente de homologação/produção usando nginx como proxy reverso,
**para que** a comunicação entre clientes e o sistema seja criptografada (FR14) sem impactar o fluxo de desenvolvimento local.

## Acceptance Criteria

1. O ambiente de homologação deve servir a aplicação via HTTPS com nginx como proxy reverso.
2. O nginx deve fazer TLS termination para o backend Go (HTTP interno) e servir o frontend Next.js.
3. O ambiente de desenvolvimento local deve continuar funcionando em HTTP sem nenhuma alteração.
4. O frontend Next.js deve ajustar as rewrites para usar HTTPS quando configurado via variável de ambiente.
5. Deve existir um `docker-compose.yml` completo que orquestre nginx + API Go + PostgreSQL para deploy.
6. O nginx deve redirecionar HTTP → HTTPS automaticamente.
7. A configuração deve suportar certificados ICP-Brasil ou Let's Encrypt.

## Dev Notes

- Ambiente de homologação usará nginx como reverse proxy (conforme decisão do time)
- Dev local permanece inalterado: `go run .` na porta 8080, `pnpm dev` no frontend
- Backend Go não precisa de alterações — nginx faz TLS termination
- Frontend Next.js: expor `NEXT_PUBLIC_API_URL` para rewrites condicionais
- nginx.conf deve tratar CORS, headers de segurança (HSTS, CSP) e logging
- Certificados: suportar ICP-Brasil (certificados A3) e Let's Encrypt (auto-renew)
- Dockerfile Go: multi-stage build (golang:1.23-alpine → distroless)
- Verificar se há porta 443 disponível no ambiente de homologação
- Referência: `docs/compliance/matriz-requisitos.md` — FR14

## Testing
