# Story 2.2.1: Extensão do Modelo de Editais

## Status

Concluída (inclui correções C-01, C-02, C-03, C-04)

## Story

**Como** equipe da FAPITEC-SE,
**quero** estender o modelo de editais com campos completos de elegibilidade, documentação e regras de submissão,
**para que** a plataforma suporte editais reais com critérios específicos por chamada.

## Context

O módulo `gestao_de_editais` atualmente possui CRUD básico (Stories 1.12 e 1.14) com os campos: Nome, Descrição, DataInício, DataFim, Status, TipoChamada, NotaDeCorte, ValorGlobal. O Termo de Referência (seção 4.1) e o ProjetoFapitec.txt (seção "Criar Edital") especificam campos adicionais necessários para editais reais da FAPITEC-SE.

## Acceptance Criteria

1. O cadastro de edital deve incluir seção "Formulário Base" com: modelo de formulário (dropdown 1-6), relatórios exigidos (múltipla escolha), nota de corte (dropdown 30/40/50).
2. O cadastro deve incluir seção "Proponente Base" com: titulação mínima para elegibilidade (Doutor, Mestre, Graduado).
3. O cadastro deve incluir seção "Empresa" com: exigir empresa vinculada (checkbox), porte permitido (MEI, ME, EPP — múltipla escolha), enquadramento (Simples Nacional, Lucro Presumido).
4. O cadastro deve incluir seção "Documentações Complementares" com checkboxes para documentos exigidos (RG, CPF, comprovante de conclusão, comprovante de residência, etc.).
5. A listagem de editais deve exibir os novos campos quando preenchidos.
6. A visualização de edital deve exibir todas as seções organizadas.
7. Os novos campos são opcionais ou têm valores default — não quebram clientes existentes.
8. A migration deve ser incremental (`004_estender_editais.sql`) sem quebrar dados existentes.
