# Story 2.1.1: Configuração do Subdomínio Casdoor no Nginx da VM

## Status

Concluído

## Story

**Como** equipe de infraestrutura da FAPITEC-SE,
**quero** configurar o nginx da Oracle Cloud VM para rotear `auth.fapitec.duckdns.org` para o serviço Casdoor (`127.0.0.1:8000`),
**para que** a UI de gestão do Casdoor e o fluxo OIDC estejam acessíveis via HTTPS com SSL válido no ambiente de homologação.

## Contexto

Referente à **Epic 2.1** (Finalização da Homologação e Pipeline CI/CD). O Casdoor IAM foi integrado e testado localmente (Story 1.18), com infraestrutura Docker e snippet nginx já criados em `docs/infra/nginx-casdoor-subdomain.conf`. Falta aplicar a configuração na VM, emitir certificado SSL e validar o fluxo OIDC ponta a ponta via subdomínio.

## Acceptance Criteria

1. O subdomínio `auth.fapitec.duckdns.org` deve responder via HTTPS com certificado SSL Let's Encrypt válido.
2. A configuração nginx deve seguir o snippet `docs/infra/nginx-casdoor-subdomain.conf`, com proxy reverso para `127.0.0.1:8000`.
3. O redirecionamento HTTP → HTTPS deve funcionar (porta 80 → 443).
4. A UI de gestão do Casdoor deve estar acessível em `https://auth.fapitec.duckdns.org` com login admin funcional.
5. O fluxo OIDC completo deve funcionar: login → redirect para Casdoor → callback → JWT válido.
6. A configuração não deve afetar as rotas existentes do nginx (`/api/` → API Go, demais → Next.js).
7. As variáveis de ambiente em `.env.homolog` devem ser atualizadas com o endpoint correto do Casdoor (`https://auth.fapitec.duckdns.org`).
8. Headers de segurança (HSTS, X-Content-Type-Options) devem estar presentes na resposta.
9. O certificado SSL deve ter auto-renew configurado via cron (certbot).
