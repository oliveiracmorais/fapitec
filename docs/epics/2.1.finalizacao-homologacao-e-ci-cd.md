# Epic 2.1: Finalização da Homologação e Pipeline CI/CD

## Status

Concluído

## Epic Goal

Finalizar a configuração do ambiente de homologação (Casdoor IAM com nginx), validar a resiliência do sistema de identidade e acesso, e automatizar o pipeline de deploy para reduzir risco operacional e acelerar entregas futuras.

## Epic Description

**Existing System Context:**

O ambiente de homologação está operacional na Oracle Cloud VM (ARM64, Docker Compose) com Next.js, API Go, PostgreSQL 16, Casdoor e Dozzle. O Casdoor IAM (Story 1.18) foi integrado e testado localmente, mas o subdomínio `auth.fapitec.duckdns.org` ainda não foi configurado no nginx da VM.

**Enhancement Details:**

Esta Epic entrega três melhorias sequenciais no ambiente de homologação:

- [x] Feature flag `AUTH_PROVIDER` mantém compatibilidade retroativa

## Risk Mitigation

| Risco | Mitigação |
|-------|-----------|
| SSL misconfig expõe serviço | Validar certificado pós-deploy; usar staging Let's Encrypt primeiro |
| Fallback internal quebra com dados do Casdoor | Testar migração reversa em ambiente isolado antes da homologação |
| CI/CD corrompe ambiente estável | Healthcheck pós-deploy + rollback automático; deploy em horário programado |

**Rollback Plan:**
- Nginx Casdoor: restaurar configuração anterior do nginx + recarregar (`nginx -s reload`)
- Testes de resiliência: são não-destrutivos por definição (rollback = restaurar `.env` anterior e reiniciar container)
- CI/CD: botão de rollback via `revert` commit ou re-executar pipeline com versão anterior
